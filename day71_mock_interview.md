# 模拟面试 + 复盘（Day71）

> 用法：找人或自己录音，按下面流程走一遍 45 分钟，录下来回放。目标不是答对，是找出**3 个讲不清的薄弱点**，回到对应阶段补。讲不清 = 还没学透。

## 模拟面试流程（45 分钟）

### 1. 自我介绍 + 项目讲解（5 分钟）
按 `项目讲解_5分钟.md` 讲。录下来回放，检查：有没有结论先行？有没有报菜名？数字是不是张口就来？

### 2. 项目深挖（15 分钟，面试官顺着追）
预期被追的点（每个都要能往下接两层）：
- "权限为什么不能靠 prompt？" → 注入泄露 → 检索层 `can_see` 怎么实现。
- "评测集怎么造的？拒答题怎么测？" → schema → 拒答正确率怎么算。
- "CI 门禁怎么拦的？阈值怎么定的？" → `ci_gate.py` exit code → 阈值进 git 可回溯。
- "成本优化省了多少？质量掉了吗？" → 缓存 + 路由 → 评测验证没塌的那组数。

### 3. 八股 + 场景题（20 分钟）
从两份题库抽：`面试题准备.md`（RAG/Agent/选型）+ `面试题_企业与上线.md`（可靠性/权限/多租户/CI/上线）。重点抽这些高频：
- RAG 答错怎么诊断是检索还是生成？
- LangGraph vs Chain？状态怎么持久化？防死循环？
- 文档级权限为什么在检索层？多租户怎么强隔离？
- prompt 改了怎么回归？CI 怎么拦坏改动？
- RAG vs 微调怎么选？怎么降幻觉？

### 4. 反问环节（5 分钟）
准备 2-3 个有信息量的反问（团队怎么做评测 / 上线后怎么发现幻觉 / 有没有 LLMOps 流程），别问"加不加班"。
