> 定位主线：**测试工程师 → AI 质量/应用工程**。差异化卖点是"会做可测试、可评测、可回归、可上线的 LLM 系统"，而不是只会调 API。

## 项目描述（可直接放简历）

**企业知识库 Agent 与自动化评测平台**

基于 LangChain + LangGraph 构建企业文档问答 Agent，支持 PDF/Markdown 解析、多文档检索、引用溯源、多轮对话。

- 向量检索 + metadata 过滤 + 混合检索（向量 + BM25）改善专有名词召回；Chroma 持久化。
- LangGraph 实现状态持久化（checkpoint）、工具调用与 human-in-the-loop 审批节点；高风险动作必过人工。
- 构建评测集作为回归用例库，覆盖事实 / 跨文档 / 拒答 / 引用准确性；用拒答正确率量化防幻觉能力。
- 接入 RAGAS / LangSmith 区分检索失败与生成幻觉；pytest 自动化回归，每次改动一条命令跑全集。
- FastAPI 服务化 + SQLite 落库 + 结构化日志；Docker 打包部署。

## 护城河叙事（harness engineering）

> 行业共识：**Agent = Model + Harness**——模型是 agent 系统里最小的部分，harness（工具调用解析、记忆、可靠性、安全、评测）才决定生产可靠性。

把自己定位成"做 harness 的人"：测试 + 工程 + 评测正是 harness engineering 的核心。比单说"会 RAG"有辨识度。评测集 = agentic loop 的"反馈信号"，把测试背景和 Agent 质量显式挂钩。
